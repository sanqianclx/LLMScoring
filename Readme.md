项目需求：基于LLM的自动化评分教学系统
一、核心目标
开发一个面向教师的轻量化在线测评系统，支持填空题、简答题的自动评分。系统利用大语言模型（LLM）对学生答案进行知识点匹配，依据教师预设的关键点自动评分，并输出评语与评分依据，最终支持教师审核与调整。
二、用户角色与功能流程
1. 教师端功能
   账户管理：
   注册/登录（需填写：学校、所教课程）。
   管理个人资料。
   课程与试卷管理：
   创建课程，为课程添加试卷。
   为试卷添加题目（支持填空、简答）并设定：
   题目文本
   参考答案（标注关键词/得分点及分值）
   总分值
   查看、编辑、删除已创建的试卷。
   试卷分享：
   生成唯一“分享代码”，学生凭此代码可访问试卷（无答案）。
   评分与审核：
   查看学生提交的答案、LLM自动评分结果、评语及评分依据。
   支持教师手动修改每题的评分与评语。
   审核确认后，学生方可查看评分结果。
   数据查看：
   支持按课程、试卷、学生查看历史评分记录。
2. 学生端功能
   免登录访问：
   通过教师分享的代码进入试卷页面。
   答题时需填写学号（必填）。
   答题与提交：
   在线填写答案（支持文本输入）。
   提交后答案发送至LLM评分。
   结果查看：
   提交后显示“已提交，等待评分”。
   教师审核通过后，凭同一分享代码查看：
   每道题的评分、评语、评分依据
   试卷总分
   三、自动化评分模块（LLM集成）
   输入信息：
   题目文本
   学生答案
   教师预设的关键词/得分点（含分值）
   课程名称（用于 contextual 评分）
   LLM处理要求：
   知识点匹配：提取学生答案中的知识点，与教师设定的关键点进行匹配。
   评分输出：
   每题得分（按点给分）
   评语（针对答案的准确性、完整性、表达清晰度等）
   评分依据（列出匹配到的关键点及扣分原因）
   总分计算：自动汇总每题得分。
   评分约束：
   填空题：精确匹配与语义匹配结合。
   简答题：按关键点覆盖率、逻辑性、表述质量综合评分。
   四、系统架构建议
   前端：简易响应式界面（可考虑Vue/React）
   后端：用户管理、试卷存储、答案提交、LLM API 调用
   数据库：存储教师、课程、试卷、学生答案、评分结果
   LLM服务：通过API调用（如DeepSeek、GPT等），需设计结构化prompt确保评分一致性。
   五、输出要求
   每题评分结果（得分/总分）
   评语（鼓励性、指导性）
   评分依据（具体说明匹配情况）
   总分及试卷级别反馈
   教师可覆盖任何自动评分结果
   六、扩展性考虑
   未来可支持选择题、判断题
   支持批量导出成绩
   可增加多教师协同管理课程
   可考虑加入查重分析（答案重复检测）
