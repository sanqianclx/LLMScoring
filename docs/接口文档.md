# 接口文档

本文档描述当前后端 REST API。接口统一由 Spring Boot 提供，基础路径为 `/api`。

## 1. 基本说明

- 协议：HTTP
- 数据格式：`application/json`
- 认证方式：当前版本为演示型实现，教师登录后由前端保存教师 ID，接口本身未接入正式 Token 认证
- 错误处理：接口异常会返回 HTTP 4xx/5xx，并附带错误消息

## 2. 接口列表

### 2.1 系统初始化

#### `GET /api/bootstrap`
作用：获取系统初始化展示信息。

响应示例字段：
- `message`：初始化提示信息
- `demoTeacher`：演示教师信息
- `demoShareCode`：演示试卷分享码
- `teachers`：当前系统中的教师列表

### 2.2 教师账号相关

#### `POST /api/teachers/register`
作用：注册教师账号。

请求体：
```json
{
  "name": "张老师",
  "username": "zhang@tju.edu.cn",
  "password": "123456",
  "school": "天津大学",
  "taughtCourse": "微积分"
}
```

字段说明：
- `name`：教师姓名
- `username`：用户名，当前也被前端当作邮箱使用
- `password`：密码
- `school`：所属学校
- `taughtCourse`：任教课程，当前为逗号分隔字符串

响应体：
```json
{
  "id": "uuid",
  "name": "张老师",
  "username": "zhang@tju.educn",
  "school": "天津大学",
  "taughtCourse": "微积分"
}
```

常见错误：
- `409 Conflict`：用户名已存在
- `400 Bad Request`：必填项为空

#### `POST /api/teachers/login`
作用：教师登录。

请求体：
```json
{
  "username": "teacher",
  "password": "teacher123"
}
```

响应体：返回完整教师工作台数据，结构与 `GET /api/teachers/{teacherId}/dashboard` 相同。

常见错误：
- `401 Unauthorized`：用户名或密码错误

#### `GET /api/teachers/{teacherId}/dashboard`
作用：获取教师工作台数据。

路径参数：
- `teacherId`：教师 ID

响应体结构：
```json
{
  "teacher": { "id": "uuid", "name": "林老师", "username": "teacher", "school": "星火中学", "taughtCourse": "高中生物" },
  "courses": [],
  "papers": [],
  "submissions": [],
  "stats": {
    "courseCount": 1,
    "paperCount": 1,
    "submissionCount": 3,
    "reviewedCount": 1,
    "pendingCount": 2
  }
}
```

#### `PUT /api/teachers/{teacherId}`
作用：修改教师资料。

路径参数：
- `teacherId`：教师 ID

请求体：
```json
{
  "name": "林老师",
  "school": "星火中学",
  "taughtCourse": "高中生物, 竞赛生物"
}
```

说明：所有字段均可选，不传则保持原值。

响应体：返回修改后的教师信息。

### 2.3 课程管理

#### `POST /api/teachers/{teacherId}/courses`
作用：创建课程。

路径参数：
- `teacherId`：教师 ID

请求体：
```json
{
  "name": "高中生物核心知识",
  "description": "围绕细胞结构与光合作用的核心单元。"
}
```

响应体：
```json
{
  "id": "uuid",
  "teacherId": "uuid",
  "name": "高中生物核心知识",
  "description": "围绕细胞结构与光合作用的核心单元。",
  "createdAt": "2026-04-07T10:00:00Z"
}
```

#### `DELETE /api/teachers/{teacherId}/courses/{courseId}`
作用：删除课程。

路径参数：
- `teacherId`：教师 ID
- `courseId`：课程 ID

响应：
- 成功时无响应体

常见错误：
- `404 Not Found`：课程不存在或不属于该教师
- `409 Conflict`：课程下仍存在试卷，不能直接删除

### 2.4 试卷管理

#### `POST /api/teachers/{teacherId}/papers`
作用：创建试卷。

路径参数：
- `teacherId`：教师 ID

请求体：
```json
{
  "courseId": "uuid",
  "title": "光合作用单元测评",
  "description": "用于章节练习",
  "active": true,
  "questions": [
    {
      "type": "FILL_BLANK",
      "text": "植物细胞中主要进行光合作用的细胞器是什么？",
      "referenceAnswer": "叶绿体",
      "maxScore": 10,
      "scoringPoints": [
        {
          "keyword": "叶绿体|chloroplast",
          "score": 10,
          "description": "指出叶绿体"
        }
      ]
    }
  ]
}
```

字段说明：
- `courseId`：所属课程 ID
- `title`：试卷标题
- `description`：试卷描述
- `active`：是否开放给学生访问
- `questions`：题目列表
- `questions[].type`：题型，`FILL_BLANK` 或 `SHORT_ANSWER`
- `questions[].text`：题目内容
- `questions[].referenceAnswer`：参考答案
- `questions[].maxScore`：题目分值
- `questions[].scoringPoints`：评分点列表

响应体：返回保存后的完整试卷结构。

#### `PUT /api/teachers/{teacherId}/papers/{paperId}`
作用：更新试卷。

路径参数：
- `teacherId`：教师 ID
- `paperId`：试卷 ID

请求体：与创建试卷相同。

响应体：返回更新后的试卷数据。

#### `DELETE /api/teachers/{teacherId}/papers/{paperId}`
作用：删除试卷。

路径参数：
- `teacherId`：教师 ID
- `paperId`：试卷 ID

响应：
- 成功时无响应体

说明：当前实现会先删除该试卷下的提交记录，再删除试卷本身。

### 2.5 学生答题相关

#### `GET /api/student/papers/{shareCode}`
作用：学生通过分享码获取试卷。

路径参数：
- `shareCode`：试卷分享码

响应体：
```json
{
  "id": "uuid",
  "courseId": "uuid",
  "title": "光合作用单元测评",
  "description": "用于章节练习",
  "shareCode": "BIO-2026",
  "questions": [
    {
      "id": "uuid",
      "type": "FILL_BLANK",
      "text": "植物细胞中主要进行光合作用的细胞器是什么？",
      "maxScore": 10
    }
  ]
}
```

常见错误：
- `404 Not Found`：分享码无效或试卷未开放

#### `POST /api/student/papers/{shareCode}/submissions`
作用：学生提交答卷，并触发自动评分。

路径参数：
- `shareCode`：试卷分享码

请求体：
```json
{
  "studentId": "2026001",
  "studentName": "张三",
  "answers": [
    {
      "questionId": "uuid",
      "answerText": "叶绿体"
    }
  ]
}
```

响应体：
- 返回本次提交记录
- 包含 `autoScores`、`finalScores`、`autoTotal`、`finalTotal`、`overallFeedback`、`status`

说明：
- 初次提交后，`status` 通常为 `PENDING_REVIEW`
- `autoScores` 是自动评分结果
- `finalScores` 在审核前与自动评分保持一致

#### `GET /api/student/results/{shareCode}?studentId=xxx`
作用：学生查看结果。

路径参数：
- `shareCode`：试卷分享码

查询参数：
- `studentId`：学生学号

响应体：
```json
{
  "message": "Review finished. Final scores are now available.",
  "status": "REVIEWED",
  "totalScore": 30.0,
  "overallFeedback": "整体掌握较好……",
  "paper": { },
  "scores": [  ]
}
```

说明：
- 若教师尚未审核，则 `scores` 可能为空，并通过 `message` 提示待审核状态
- 若已审核，则返回每题得分、评语和评分依据

### 2.6 教师评分审核

#### `PUT /api/teachers/{teacherId}/submissions/{submissionId}/review`
作用：教师审核学生答卷并给出最终评分。

路径参数：
- `teacherId`：教师 ID
- `submissionId`：提交记录 ID

请求体：
```json
{
  "questionReviews": [
    {
      "questionId": "uuid",
      "score": 10,
      "comment": "填空准确",
      "rationale": "答案与标准答案一致"
    }
  ],
  "overallFeedback": "审核通过，知识点覆盖完整。"
}
```

字段说明：
- `questionReviews`：逐题审核列表
- `questionReviews[].questionId`：题目 ID
- `questionReviews[].score`：最终得分
- `questionReviews[].comment`：教师评语
- `questionReviews[].rationale`：评分依据说明
- `overallFeedback`：整体评语

响应体：
- 返回审核后的完整提交记录
- `status` 会更新为 `REVIEWED`
- `reviewedAt` 会写入审核时间

常见错误：
- `404 Not Found`：提交记录不存在
- `403 Forbidden`：该提交不属于当前教师

## 3. 主要响应对象说明

### 3.1 `TeacherDashboardView`
用于教师工作台首页。

包含字段：
- `teacher`：教师基本信息
- `courses`：课程列表
- `papers`：试卷列表
- `submissions`：提交记录列表
- `stats`：统计信息

### 3.2 `SubmissionView`
用于教师审核页和提交流程。

包含字段：
- `id`：提交记录 ID
- `paperId`：试卷 ID
- `teacherId`：教师 ID
- `courseId`：课程 ID
- `shareCode`：分享码
- `studentId`：学号
- `studentName`：姓名
- `answers`：学生答案列表
- `autoScores`：自动评分结果
- `finalScores`：最终评分结果
- `autoTotal`：自动评分总分
- `finalTotal`：最终评分总分
- `overallFeedback`：整体评语
- `status`：状态
- `submittedAt`：提交时间
- `reviewedAt`：审核时间

### 3.3 `QuestionScoreView`
用于展示单题评分结果。

包含字段：
- `questionId`：题目 ID
- `score`：得分
- `maxScore`：满分
- `matchedPoints`：命中的得分点
- `missingPoints`：缺失的得分点
- `comment`：评语
- `rationale`：评分依据
- `overridden`：是否被教师覆盖修改

## 4. 调用顺序建议

### 教师端典型流程
1. 调用 `POST /api/teachers/login` 登录。
2. 调用 `GET /api/teachers/{teacherId}/dashboard` 刷新工作台。
3. 调用 `POST /api/teachers/{teacherId}/courses` 创建课程。
4. 调用 `POST /api/teachers/{teacherId}/papers` 创建试卷。
5. 把返回的 `shareCode` 发给学生。
6. 学生提交后，教师调用 `PUT /api/teachers/{teacherId}/submissions/{submissionId}/review` 完成审核。

### 学生端典型流程
1. 调用 `GET /api/student/papers/{shareCode}` 获取试卷。
2. 调用 `POST /api/student/papers/{shareCode}/submissions` 提交答案。
3. 调用 `GET /api/student/results/{shareCode}?studentId=...` 查看结果。
