# 接口文档

本文档说明当前版本后端 REST API 的接口设计。系统后端基于 Spring Boot 实现，统一基础路径为 `/api`。

## 1. 基本说明

### 1.1 协议与数据格式

- 协议：HTTP
- 数据格式：`application/json`
- 时间字段：使用 ISO-8601 时间字符串
- 主键字段：当前均使用 UUID

### 1.2 认证与权限

- 当前版本仍为演示型实现
- 教师登录后，前端保存教师 ID 并继续请求接口
- 后端尚未接入正式 Token / Session 认证机制
- 教师审核接口会校验提交记录是否属于当前教师

### 1.3 错误处理

- 参数错误通常返回 `400 Bad Request`
- 未登录或登录信息错误通常返回 `401 Unauthorized`
- 数据不存在通常返回 `404 Not Found`
- 资源冲突通常返回 `409 Conflict`
- LLM 评分未正确配置且禁止回退时，可能返回 `500` 或 `502`

### 1.4 自动评分说明

当前自动评分模块支持两种运行方式：

- 启发式评分：基于关键词匹配、得分点覆盖率和文本相似度
- LLM 评分：通过 OpenAI-compatible `/chat/completions` 接口完成评分

评分选择逻辑：

1. 若 `app.llm.enabled=false`，则直接使用启发式评分
2. 若启用了 LLM 且配置完整，则优先尝试 LLM 评分
3. 若启用了 LLM，但配置不完整或调用失败，且允许回退，则回退到启发式评分

## 3. 接口列表

### 3.1 系统初始化

#### `GET /api/bootstrap`

作用：获取系统初始化展示信息。

说明：

- 用于前端首次加载时展示提示语、演示教师和演示分享码
- 演示教师和演示试卷不是通过 SQL 脚本插入，而是在应用启动时若库中不存在 `teacher` 用户，则自动 seed

响应示例：

```json
{
  "message": "已加载本地开发演示账户。",
  "demoTeacher": {
    "id": "uuid",
    "name": "演示教师",
    "username": "teacher",
    "school": "演示学校",
    "taughtCourse": "生物学"
  },
  "demoShareCode": "BIO-2026",
  "teachers": [
    {
      "id": "uuid",
      "name": "演示教师",
      "username": "teacher",
      "school": "演示学校",
      "taughtCourse": "生物学"
    }
  ]
}
```

### 3.2 教师账号相关

#### `POST /api/teachers/register`

作用：注册教师账号。

请求体：

```json
{
  "name": "张老师",
  "username": "zhang@tju.edu.cn",
  "password": "123456",
  "school": "天津大学",
  "taughtCourse": "微积分, 高等代数"
}
```

字段说明：

- `name`：教师姓名，必填
- `username`：登录用户名，前端当前也将其作为邮箱展示
- `password`：密码，必填
- `school`：所属学校，必填
- `taughtCourse`：任教课程描述，当前以逗号分隔字符串保存

成功响应：

```json
{
  "id": "uuid",
  "name": "张老师",
  "username": "zhang@tju.edu.cn",
  "school": "天津大学",
  "taughtCourse": "微积分, 高等代数"
}
```

常见错误：

- `409 Conflict`：用户名已存在
- `400 Bad Request`：必填项为空

#### `POST /api/teachers/login`

作用：教师登录。

请求体：

```json
{
  "username": "teacher",
  "password": "teacher123"
}
```

响应说明：

- 成功后直接返回完整教师工作台数据
- 响应结构与 `GET /api/teachers/{teacherId}/dashboard` 一致

常见错误：

- `401 Unauthorized`：用户名或密码错误

#### `GET /api/teachers/{teacherId}/dashboard`

作用：获取教师工作台数据。

路径参数：

- `teacherId`：教师 ID

响应示例：

```json
{
  "teacher": {
    "id": "uuid",
    "name": "演示教师",
    "username": "teacher",
    "school": "演示学校",
    "taughtCourse": "生物学"
  },
  "courses": [],
  "papers": [],
  "submissions": [],
  "stats": {
    "courseCount": 1,
    "paperCount": 1,
    "submissionCount": 3,
    "reviewedCount": 1,
    "pendingCount": 2
  }
}
```

#### `PUT /api/teachers/{teacherId}`

作用：修改教师资料。

路径参数：

- `teacherId`：教师 ID

请求体：

```json
{
  "name": "林老师",
  "school": "星火中学",
  "taughtCourse": "高中生物, 竞赛生物"
}
```

说明：

- `name`、`school`、`taughtCourse` 均可选
- 不传的字段保持原值

### 3.3 课程管理

#### `POST /api/teachers/{teacherId}/courses`

作用：创建课程。

请求体：

```json
{
  "name": "高中生物核心知识",
  "description": "围绕细胞结构与光合作用的核心单元。"
}
```

成功响应：

```json
{
  "id": "uuid",
  "teacherId": "uuid",
  "name": "高中生物核心知识",
  "description": "围绕细胞结构与光合作用的核心单元。",
  "createdAt": "2026-04-07T10:00:00Z"
}
```

#### `DELETE /api/teachers/{teacherId}/courses/{courseId}`

作用：删除课程。

说明：

- 仅允许删除属于当前教师的课程
- 若课程下仍存在试卷，则不允许删除

常见错误：

- `404 Not Found`：课程不存在或不属于当前教师
- `409 Conflict`：课程下仍存在试卷

### 3.4 试卷管理

#### `POST /api/teachers/{teacherId}/papers`

作用：创建试卷。

请求体：

```json
{
  "courseId": "uuid",
  "title": "光合作用单元测评",
  "description": "用于章节练习",
  "active": true,
  "questions": [
    {
      "type": "FILL_BLANK",
      "text": "植物细胞中主要进行光合作用的细胞器是什么？",
      "referenceAnswer": "叶绿体",
      "maxScore": 10,
      "scoringPoints": [
        {
          "keyword": "叶绿体|chloroplast",
          "score": 10,
          "description": "指出叶绿体"
        }
      ]
    },
    {
      "type": "SHORT_ANSWER",
      "text": "简述光合作用的条件与产物。",
      "referenceAnswer": "光合作用需要光照、水和二氧化碳，并产生葡萄糖和氧气。",
      "maxScore": 20,
      "scoringPoints": [
        {
          "keyword": "光照|阳光",
          "score": 4,
          "description": "提到光照"
        },
        {
          "keyword": "二氧化碳|CO2",
          "score": 4,
          "description": "提到二氧化碳"
        }
      ]
    }
  ]
}
```

字段说明：

- `courseId`：所属课程 ID
- `title`：试卷标题，必填
- `description`：试卷说明
- `active`：是否开放给学生访问，不传时默认 `true`
- `questions`：题目列表，至少一题
- `questions[].type`：题型，当前支持 `FILL_BLANK` 和 `SHORT_ANSWER`
- `questions[].text`：题目内容
- `questions[].referenceAnswer`：参考答案
- `questions[].maxScore`：题目分值，必须大于 0
- `questions[].scoringPoints`：评分点列表

#### `PUT /api/teachers/{teacherId}/papers/{paperId}`

作用：更新试卷。

说明：

- 请求体与创建试卷相同
- 更新时保留原有分享码

#### `DELETE /api/teachers/{teacherId}/papers/{paperId}`

作用：删除试卷。

说明：

- 当前实现会先删除该试卷下的提交记录，再删除试卷本身

### 3.5 学生答题相关

#### `GET /api/student/papers/{shareCode}`

作用：学生通过分享码获取试卷。

说明：

- 仅当试卷存在且 `active=true` 时可访问
- 学生视图不返回参考答案和评分点

响应示例：

```json
{
  "id": "uuid",
  "courseId": "uuid",
  "title": "光合作用单元测评",
  "description": "用于章节练习",
  "shareCode": "BIO-2026",
  "questions": [
    {
      "id": "uuid",
      "type": "FILL_BLANK",
      "text": "植物细胞中主要进行光合作用的细胞器是什么？",
      "maxScore": 10.0
    }
  ]
}
```

常见错误：

- `404 Not Found`：分享码无效或试卷未开放

#### `POST /api/student/papers/{shareCode}/submissions`

作用：学生提交答卷，并触发自动评分。

请求体：

```json
{
  "studentId": "2026001",
  "studentName": "张三",
  "answers": [
    {
      "questionId": "uuid",
      "answerText": "叶绿体"
    }
  ]
}
```

说明：

- `studentId` 必填
- `studentName` 可为空，后端会使用默认占位名
- 若某题未提交答案，系统会按空串参与评分

响应特点：

- 返回完整 `SubmissionView`
- 初次提交后 `status` 为 `PENDING_REVIEW`
- `autoScores` 为自动评分结果
- 审核前 `finalScores` 与 `autoScores` 相同
- `overallFeedback` 由评分服务自动生成

#### `GET /api/student/results/{shareCode}?studentId=xxx`

作用：学生按分享码和学号查看当前结果。

查询参数：

- `studentId`：学生学号

返回逻辑：

- 若教师尚未审核：
  - `status = PENDING_REVIEW`
  - `totalScore = 0`
  - `scores = []`
  - `message` 提示等待教师发布成绩
- 若教师已审核：
  - `status = REVIEWED`
  - `totalScore` 为最终分数
  - `scores` 返回每题得分、评语和评分依据

已审核响应示例：

```json
{
  "message": "Review finished. Final scores are now available.",
  "status": "REVIEWED",
  "totalScore": 30.0,
  "overallFeedback": "整体掌握较好，知识点覆盖完整。",
  "paper": {
    "id": "uuid",
    "courseId": "uuid",
    "title": "光合作用单元测评",
    "description": "用于章节练习",
    "shareCode": "BIO-2026",
    "questions": []
  },
  "scores": [
    {
      "questionId": "uuid",
      "score": 10.0,
      "maxScore": 10.0,
      "matchedPoints": ["叶绿体"],
      "missingPoints": [],
      "comment": "填空准确",
      "rationale": "答案与标准答案一致",
      "overridden": true
    }
  ]
}
```

### 3.6 教师评分审核

#### `PUT /api/teachers/{teacherId}/submissions/{submissionId}/review`

作用：教师审核学生答卷并发布最终成绩。

请求体：

```json
{
  "questionReviews": [
    {
      "questionId": "uuid",
      "score": 10,
      "comment": "填空准确",
      "rationale": "答案与标准答案一致"
    }
  ],
  "overallFeedback": "审核通过，知识点覆盖完整。"
}
```

字段说明：

- `questionReviews`：逐题审核列表
- `questionReviews[].questionId`：题目 ID
- `questionReviews[].score`：最终得分，后端会按该题满分做截断
- `questionReviews[].comment`：教师评语
- `questionReviews[].rationale`：评分依据说明
- `overallFeedback`：整体评语；若不传，则由系统按总分自动生成

响应说明：

- 返回审核后的完整提交记录
- `status` 更新为 `REVIEWED`
- `reviewedAt` 写入审核时间
- 被审核覆盖的题目其 `overridden=true`

常见错误：

- `404 Not Found`：提交记录不存在
- `403 Forbidden`：该提交不属于当前教师

## 4. 接口调用顺序建议

### 4.1 教师端典型流程

1. 调用 `POST /api/teachers/login` 登录
2. 调用 `GET /api/teachers/{teacherId}/dashboard` 获取工作台数据
3. 调用 `POST /api/teachers/{teacherId}/courses` 创建课程
4. 调用 `POST /api/teachers/{teacherId}/papers` 创建试卷
5. 将 `shareCode` 发给学生
6. 学生提交后，教师调用 `PUT /api/teachers/{teacherId}/submissions/{submissionId}/review` 完成审核

### 4.2 学生端典型流程

1. 调用 `GET /api/student/papers/{shareCode}` 获取试卷
2. 调用 `POST /api/student/papers/{shareCode}/submissions` 提交答案
3. 调用 `GET /api/student/results/{shareCode}?studentId=...` 查询结果

## 5. 当前实现备注

### 5.1 关于演示数据

- 默认演示教师：`teacher / teacher123`
- 默认演示分享码：`BIO-2026`
- 这些数据是在应用启动时自动 seed 的，不属于 `schema.sql` 或 `mysql-init.sql` 的静态插入内容

### 5.2 关于忘记密码

- 前端保留了忘记密码页面入口
- 当前后端未提供找回密码接口
- 该功能仍属于预留状态

### 5.3 关于接口范围

当前版本接口主要覆盖：

- 教师注册与登录
- 教师资料维护
- 课程管理
- 试卷管理
- 学生答题
- 教师审核
- 学生成绩查询

尚未提供：

- 正式认证接口
- 文件导入导出接口
- 统计分析接口
- 系统管理接口
