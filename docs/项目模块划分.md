# 项目模块划分

本文档根据原始需求说明整理，目的是把“项目要做什么”拆成清晰的功能模块，便于汇报、开发分工和后续迭代。

## 1. 项目目标

本项目为“基于 LLM 的自动评分教学系统”。系统面向教师和学生两类用户，支持教师创建课程与试卷，学生通过分享码在线答题，系统对填空题与简答题进行自动评分，并允许教师对结果进行人工审核、修改和发布。

## 2. 总体模块划分

项目可以划分为 7 个一级模块：
1. 教师账号与个人信息模块
2. 课程管理模块
3. 试卷管理与分享模块
4. 学生答题模块
5. 自动评分模块
6. 教师审核与结果发布模块
7. 数据存储与系统支撑模块

## 3. 各模块详细说明

### 3.1 教师账号与个人信息模块

功能目标：
- 支持教师注册与登录
- 支持教师维护个人信息
- 支持记录教师所属学校和任教课程

核心功能：
- 教师注册
- 教师登录
- 教师资料查看与修改
- 教师基本身份信息维护

输入信息：
- 教师姓名
- 用户名或邮箱
- 登录密码
- 所属学校
- 任教课程

输出结果：
- 教师基础账户信息
- 教师工作台入口

### 3.2 课程管理模块

功能目标：
- 让教师按课程组织试卷
- 支持课程信息的新增、查看和删除

核心功能：
- 创建课程
- 查看课程列表
- 删除课程
- 通过课程归类试卷

输入信息：
- 课程名称
- 课程描述
- 所属教师

输出结果：
- 课程列表
- 课程详情数据

### 3.3 试卷管理与分享模块

功能目标：
- 支持教师创建和维护试卷
- 支持为试卷配置题目、参考答案和评分点
- 支持为试卷生成分享码，供学生进入答题

核心功能：
- 创建试卷
- 编辑试卷
- 删除试卷
- 添加题目
- 配置评分点
- 控制试卷开放状态
- 生成试卷分享码

题目支持：
- 填空题
- 简答题

每道题的配置内容：
- 题目文本
- 参考答案
- 关键词或得分点
- 每个得分点的分值
- 题目总分

输出结果：
- 试卷详情
- 题目列表
- 分享码
- 学生可访问的试卷视图

### 3.4 学生答题模块

功能目标：
- 学生无需注册即可答题
- 通过教师发放的分享码访问试卷
- 填写学号后完成答题和提交

核心功能：
- 根据分享码获取试卷
- 填写学号和姓名
- 在线作答
- 提交答卷
- 查询答题结果

输入信息：
- 分享码
- 学号
- 姓名（可选）
- 每道题的答案内容

输出结果：
- 提交成功提示
- 待审核提示
- 已审核后的每题得分与总分

### 3.5 自动评分模块

功能目标：
- 对学生提交的答案进行自动评分
- 根据教师预设的评分点生成每题得分、评语和评分依据

输入信息：
- 题目文本
- 参考答案
- 学生答案
- 教师设置的关键词和得分点
- 每个得分点的分值
- 课程名称

核心处理逻辑：
- 知识点匹配
- 关键词匹配
- 语义相似度判断
- 分数累计
- 每题评语生成
- 总体评语生成

评分输出内容：
- 每题得分
- 每题满分
- 命中的得分点
- 缺失的得分点
- 单题评语
- 评分依据
- 自动评分总分

评分约束：
- 填空题偏向精确匹配与近义匹配结合
- 简答题偏向得分点覆盖率、语义接近度和内容完整性

### 3.6 教师审核与结果发布模块

功能目标：
- 自动评分后，教师可对结果进行人工审核
- 教师审核通过后，学生才能查看最终成绩

核心功能：
- 查看学生提交记录
- 查看系统自动评分结果
- 查看命中点与失分点
- 修改单题得分
- 修改单题评语
- 填写整体评语
- 提交最终审核结果

输入信息：
- 自动评分结果
- 教师修正后的单题得分
- 教师修正后的评语
- 教师整体评价

输出结果：
- 最终评分结果
- 已审核状态
- 学生可查看的正式结果

### 3.7 数据存储与系统支撑模块

功能目标：
- 持久化保存教师、课程、试卷、题目、评分点、学生提交和评分结果
- 为前端和业务模块提供基础支撑

当前数据对象：
- 教师
- 课程
- 试卷
- 题目
- 题目评分点
- 学生提交
- 学生答案
- 自动评分结果
- 最终评分结果
- 得分点明细

系统支撑内容：
- REST API 接口
- JPA 实体映射
- MySQL 数据库存储
- Vue 前端访问
- 前后端路由协同

## 4. 角色视角下的功能链路

### 4.1 教师端业务链路
1. 注册或登录教师账号。
2. 进入教师工作台。
3. 创建课程。
4. 在课程下创建试卷。
5. 为试卷配置题目、答案与评分点。
6. 生成分享码并发给学生。
7. 查看学生提交。
8. 查看自动评分结果。
9. 审核并发布最终成绩。

### 4.2 学生端业务链路
1. 获取教师发放的分享码。
2. 进入答题页面。
3. 输入学号和答案。
4. 提交试卷。
5. 等待教师审核。
6. 返回结果页查看最终得分、评语和评分依据。

## 5. 系统架构对应关系

从实现角度看，项目可以对应为以下技术模块：

### 前端模块
- 登录注册页面
- 教师工作台页面
- 课程管理页面
- 试卷编辑页面
- 试卷分享页面
- 学生答题页面
- 教师审核页面
- 学生结果页面
- 个人中心页面

### 后端模块
- 控制器层：处理前端请求
- 服务层：处理业务逻辑
- 仓储层：提供数据库访问能力
- 实体层：映射数据库表结构
- DTO 层：封装接口传输对象

### 数据库模块
- 教师表
- 课程表
- 试卷表
- 题目表
- 题目评分点表
- 提交记录表
- 学生答案表
- 单题评分表
- 评分点明细表

## 6. 输出要求整理

系统最终需要支持输出以下结果：
- 每题得分结果
- 每题满分信息
- 每题教师评语或系统评语
- 每题评分依据
- 总分
- 总体评价
- 学生是否已审核状态
- 教师是否覆盖自动评分结果的标记

## 7. 后续扩展方向

后续可以继续扩展为：
- 支持选择题、判断题等更多题型
- 支持成绩批量导出
- 支持多个教师协同维护课程
- 支持查重检测或答案相似度分析
- 接入真实 LLM 服务替代当前启发式评分
- 接入正式权限认证和密码加密机制
