# 项目模块划分

本文档基于当前代码实现整理项目模块，用于说明“系统现在已经实现了什么”“各模块如何协作”，便于课程汇报、开发分工和后续开源迭代。

## 1. 项目目标

本项目为“基于 LLM 的自动评分教学系统”。系统面向教师和学生两类用户，围绕“教师建课建卷、学生在线答题、系统自动评分、教师复核发布成绩”的完整链路展开。

当前版本已经具备可运行的前后端原型，并支持两类自动评分方式：

- 启发式评分
- OpenAI-compatible LLM 评分

## 2. 总体模块划分

结合当前实现，项目可以划分为 8 个一级模块：

1. 教师账号与个人资料模块
2. 课程管理模块
3. 试卷管理与分享模块
4. 学生答题模块
5. 自动评分模块
6. 教师复核与成绩发布模块
7. 数据存储与系统支撑模块
8. 前端交互与页面路由模块

## 3. 各模块详细说明

### 3.1 教师账号与个人资料模块

功能目标：

- 支持教师注册与登录
- 支持教师查看和修改个人资料
- 支持维护学校信息与授课课程描述

当前已实现功能：

- 教师注册
- 教师登录
- 教师资料查看
- 教师资料修改
- 教师端默认演示账号自动初始化

输入信息：

- 教师姓名
- 用户名或邮箱
- 登录密码
- 所属学校
- 任教课程

输出结果：

- 教师基础账户信息
- 教师工作台入口

当前限制：

- 认证方式仍为演示型实现，尚未接入正式 Token / Session
- 密码仍为普通字符串存储，后续需要引入加密方案
- 忘记密码页面已预留，但尚未接入真实邮件找回流程

### 3.2 课程管理模块

功能目标：

- 让教师按课程组织试卷
- 支持课程创建、展示和删除

当前已实现功能：

- 创建课程
- 查看课程列表
- 删除课程
- 教师工作台中按课程组织试卷

输入信息：

- 课程名称
- 课程描述
- 所属教师

输出结果：

- 课程列表
- 课程详情数据

业务约束：

- 一个课程当前只归属一个教师
- 若课程下仍存在试卷，则不允许直接删除

### 3.3 试卷管理与分享模块

功能目标：

- 支持教师创建和维护试卷
- 支持配置题目、参考答案和评分点
- 支持生成分享码，供学生进入答题

当前已实现功能：

- 创建试卷
- 编辑试卷
- 删除试卷
- 添加题目
- 配置评分点
- 控制试卷开放状态
- 生成并展示试卷分享码
- 生成学生访问入口链接
- 查看试卷提交概览

当前支持题型：

- 填空题 `FILL_BLANK`
- 简答题 `SHORT_ANSWER`

每道题的配置内容：

- 题目文本
- 参考答案
- 每题满分
- 多个评分点
- 每个评分点的关键词表达式
- 每个评分点的分值
- 每个评分点的说明

输出结果：

- 试卷详情
- 题目列表
- 分享码
- 学生可访问的试卷视图

### 3.4 学生答题模块

功能目标：

- 学生无需注册即可答题
- 学生通过分享码进入试卷
- 提交后进入自动评分与教师复核流程

当前已实现功能：

- 根据分享码获取试卷
- 填写学号和姓名
- 在线作答
- 本地浏览器自动保存草稿
- 提交答卷
- 查询成绩结果

输入信息：

- 分享码
- 学号
- 姓名（可选）
- 每道题的答案内容

输出结果：

- 提交成功提示
- 待教师复核提示
- 已审核后的单题得分、总分和评语

业务说明：

- 学号为必填项
- 学生端拿到的是“脱敏试卷视图”，不会看到参考答案和评分点
- 若教师尚未审核，成绩页返回待复核状态

### 3.5 自动评分模块

功能目标：

- 对学生提交的答案进行自动评分
- 输出每题得分、评语、评分依据和总体评语

当前已实现评分路径：

- 启发式评分服务
- LLM 评分服务
- LLM 失败时回退到启发式评分

输入信息：

- 课程名称
- 题目文本
- 参考答案
- 学生答案
- 教师设置的评分点与分值

核心处理逻辑：

- 关键词匹配
- 得分点覆盖率判断
- 文本相似度判断
- 自动生成单题评语
- 自动生成整体评语
- 调用 OpenAI-compatible `/chat/completions` 接口进行 LLM 评分
- LLM 返回 JSON 后解析出分项得分、命中点和缺失点

评分输出内容：

- 每题得分
- 每题满分
- 命中的得分点
- 缺失的得分点
- 单题评语
- 评分依据
- 自动评分总分
- 整体评语

当前配置能力：

- 可通过配置项启用或关闭 LLM
- 可配置模型地址、模型名、API Key、温度、超时
- 可配置 LLM 失败时是否自动回退

### 3.6 教师复核与成绩发布模块

功能目标：

- 教师查看自动评分结果
- 教师复核并发布最终成绩

当前已实现功能：

- 查看学生提交记录
- 按状态和关键词筛选提交
- 查看学生答案
- 查看系统自动评分结果
- 查看命中点与失分点
- 修改单题最终得分
- 修改单题评语
- 填写整体评语
- 发布最终成绩

输入信息：

- 自动评分结果
- 教师修正后的单题得分
- 教师修正后的评语
- 教师修正后的整体评语

输出结果：

- 最终评分结果
- 已审核状态
- 学生可查看的正式成绩

设计特点：

- 系统会同时保留 `AUTO` 和 `FINAL` 两个评分阶段的数据
- 单题是否被教师人工覆盖会记录为 `overridden`

### 3.7 数据存储与系统支撑模块

功能目标：

- 持久化保存教师、课程、试卷、题目、评分点、学生提交和评分结果
- 为前端和业务模块提供统一数据支撑

当前数据对象：

- 教师
- 课程
- 试卷
- 题目
- 题目评分点
- 学生提交
- 学生答案
- 单题评分结果
- 命中点与失分点明细

系统支撑内容：

- REST API 接口
- JPA 实体映射
- `schema.sql` 自动建表
- `mysql-init.sql` 手动建库建表
- 启动时演示数据 seed
- 旧版本试卷 / 提交结构迁移兼容组件
- MySQL 持久化
- H2 测试环境

### 3.8 前端交互与页面路由模块

功能目标：

- 提供教师端和学生端完整页面交互
- 实现从登录到出卷、答题、复核、查分的可视化流程

当前已实现页面：

- 登录页
- 注册页
- 忘记密码占位页
- 教师工作台
- 课程管理页
- 试卷管理 / 编辑页
- 试卷分享页
- 教师评阅页
- 教师个人中心页
- 学生入口页
- 学生答题页
- 学生成绩页

前端支撑能力：

- Vue Router 路由组织
- `/api` 请求统一调用后端
- 学生答题草稿自动存储到本地浏览器
- 教师工作台、课程、试卷、提交等数据统一由前端服务层管理

## 4. 角色视角下的业务链路

### 4.1 教师端业务链路

1. 注册或登录教师账号
2. 进入教师工作台
3. 创建课程
4. 在课程下创建试卷
5. 为试卷配置题目、参考答案和评分点
6. 生成分享码并发送给学生
7. 查看学生提交
8. 查看自动评分结果
9. 复核并发布最终成绩

### 4.2 学生端业务链路

1. 获取教师发放的分享码
2. 进入学生入口页
3. 输入分享码并打开试卷
4. 填写学号和答案
5. 提交试卷
6. 等待教师审核
7. 使用分享码和学号查询最终成绩

## 5. 系统架构对应关系

从实现角度看，项目当前可以拆分为以下技术层：

### 5.1 前端层

- 登录注册页面
- 教师工作台页面
- 课程管理页面
- 试卷编辑页面
- 试卷分享页面
- 教师复核页面
- 学生入口页面
- 学生答题页面
- 学生成绩页面
- 个人中心页面

### 5.2 后端接口层

- `PlatformController`：统一暴露当前核心业务接口
- `SpaForwardController`：负责前端单页路由转发

### 5.3 后端服务层

- `PlatformService`：封装教师、课程、试卷、学生提交、教师审核等业务逻辑
- `ScoringService`：统一调度自动评分逻辑
- `HeuristicScoringService`：启发式评分实现
- `LlmScoringService`：LLM 评分实现
- `PromptTemplateService`：评分 Prompt 渲染
- `OpenAiCompatibleLlmClient`：OpenAI-compatible 接口调用

### 5.4 数据访问层

- `JpaPlatformRepository`：核心仓储适配器
- 各 JPA Repository：实体持久化访问
- 兼容迁移组件：处理旧版本结构迁移

### 5.5 数据库层

- 教师表
- 课程表
- 试卷表
- 题目表
- 题目评分点表
- 提交记录表
- 学生答案表
- 单题评分表
- 评分点明细表

## 6. 当前版本输出结果

系统当前能够输出以下关键结果：

- 每题自动评分结果
- 每题满分信息
- 命中得分点与缺失得分点
- 每题评语
- 每题评分依据
- 自动评分总分
- 教师复核后的最终单题得分
- 最终总分
- 整体评语
- 学生当前是否已审核状态
- 教师是否覆盖自动评分结果的标记

## 7. 当前版本的边界说明

为了避免文档与实现不一致，这里明确说明当前仍未完成的部分：

- 忘记密码页面仅为前端占位，未接入真实找回流程
- 认证机制仍为演示型实现，未接入正式权限体系
- 教师密码仍未加密存储
- 当前题型仅实现填空题与简答题
- 尚未实现导出成绩、批量导入、系统管理后台等增强能力

## 8. 后续扩展方向

后续可以继续扩展为：

- 支持选择题、判断题、论述题等更多题型
- 支持成绩批量导出
- 支持多个教师协同维护课程
- 支持学生表、班级表与考试场次管理
- 支持答案相似度分析与查重检测
- 接入更多 OpenAI-compatible 或本地模型服务
- 接入正式权限认证和密码加密机制
- 增加操作日志、审核历史和统计分析模块
